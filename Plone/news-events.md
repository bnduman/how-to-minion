# Adding News and Events in Plone

## What are Collections?

Collections automatically gather and display content from your site based on specific tags or criteria. Instead of manually listing items, collections update automatically whenever new content with matching tags is added.

---

## Events Collection

The events collection displays all Event items from across the site.

**Location:** https://www.dependency.uni-bonn.de/en/outreach/all-events

**Tags it catches:**
- `conference`
- `event`
- `Event`
- `Events`
- `events`
- `JCMML`
- `Lectures`

Anything tagged with these labels will automatically appear in the events collection.

## How to Add an Event

### 1. Create the Event Item

*Make sure that you are logged in*
1. Navigate to where you want to add the event
2. Click **"Add new"** â†’ **"Event"**
3. Fill in:
   - **Title:** Event name
   - **Description:** Brief summary
   - **Start date/time:** When the event begins
   - **End date/time:** When the event ends
   - **Location:** Where it takes place
   - **Body:** Full event details

### 2. Add Tags

1. At the top, switch to the **Categorization** field
2. Add one or more of the event tags:
   - `event`
   - `conference`
   - `Lectures`
   - `JCMML`
3. Make sure the tag matches exactly (case-sensitive)

Save! Always let Cecile or Laura take a look before making it public.

## Important Notes

- **Tags are case-sensitive:** `News` is different from `news`. Use the exact tags listed above.
- **Collections update automatically:** Once published with the correct tag, items appear immediately in the collection. Plone has its internal update intervals, which means the content may not appear immidietaly. You can go to edit mode on the main page and save, if you want them online asap.
- **Multiple tags:** You can add multiple tags to a single item (e.g., both `event` and `JCMML`. No reason to add both `event` and `events` though!).

---

## Troubleshooting

If your event or news doesn't appear in the collection:
1. Check that the tag is spelled correctly
2. Verify it matches the case (uppercase/lowercase) exactly
3. Make sure the item is published (not in draft)
4. Make sure the image has copyright data
